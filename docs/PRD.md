## 1. 产品概述

### 1.1 一句话描述

**一份比美食地图更精准的「肠道友好地图」—— 让肠易激人群不再因害怕找不到厕所而放弃出行。**

### 1.2 产品愿景

将一种社会功能障碍（因肠道焦虑导致的出行恐惧），转变为一个可以**管理和探索的生活选择**。

### 1.3 要解决的问题

| 问题 | 严重程度 | 波及人群 |
|---|---|---|
| 外出找不到厕所，不敢出门 | 🔴 极高 | 100% 的 IBS 患者 |
| 不知道厕所条件（蹲便/纸巾/排队） | 🔴 极高 | 90%+ |
| 无法提前规划「安全路线」 | 🟡 高 | 70%+ |
| 无法找到饮食与症状的关联 | 🟡 高 | 80%+ |
| 现有地图产品厕所信息极度粗糙 | 🟢 中 | 100% |

### 1.4 核心价值主张

> **出门前规划路线 → 沿途厕所保障 → 到地方快速找到厕所 → 随时记录身体状况 → 发现饮食规律**

一个 App 闭环解决「出行 → 应急 → 记录 → 改善」的全链路。

## 3. 用户画像

### 3.1 核心用户：王敏（30岁，北京白领）

> *"每次约朋友吃饭前，我都要先研究那个商场有几个厕所、分别在几楼。如果查不到，我就不去了。"*

**行为特征**：
- 每天出门前评估「厕所风险」
- 熟悉常去路线的所有厕所位置（比导航记得还熟）
- 避免尝试新餐厅/新路线，害怕未知
- 包里永远带纸巾和备用内裤

**核心需求**：
- 🚻 实时查看附近的可靠厕所
- 🗺️ 规划「安全」的出行路线
- 📝 记录饮食，找到 trigger food
- 🔒 隐私保护（不想让别人知道这个「毛病」）

### 3.2 次级用户：旅行者/新城市定居者

> *"刚到广州工作，完全不知道哪里能上厕所，每次出门都焦虑。"*

**核心需求**：快速了解新城市的厕所分布，建立心理安全感。

### 3.3 三级用户：关注肠道健康的普通人

> *"想记录一下肠胃规律，看看有没有什么问题。"*

**核心需求**：轻量日志 + 趋势分析，不一定需要厕所数据。

## 4. 痛点深度分析

### 痛点 1：比「找美食」更迫切的「找厕所」

**现有方案的失败**：
- 高德/百度搜「厕所」→ 结果少、信息粗糙、不准
- 大众点评 → 只有商场厕所，没有街边/公园/地铁的
- 问路人 → 尴尬、方向描述不准

**我们的解法**：
- 众包 + 政府数据 + OSM → **比美食地图更详细的厕所 POI**
- 维度：蹲便/坐便、纸巾、排队时间、开放时间、收费、无障碍、清洁度
- 每个厕所都是「双向信息」：官方数据 + 用户实时验证

### 痛点 2：「肠道焦虑」是一种社交障碍

**心理机制**：
```
害怕找不到厕所 → 减少外出 → 社交活动减少 → 孤独/抑郁
                → 拒绝旅行   → 生活质量下降
                → 不敢尝试新事物 → 人生半径缩小
```

**我们的解法**：
- 路线规划 → 把「不确定性」变成「可控风险」
- 安全评分 → 可视化焦虑，量化安全感
- 社区评价 → 用户间的互助感，「我不是一个人」

### 痛点 3：找不到饮食与症状的关联

**现状**：多数 IBS 患者靠「感觉」判断 trigger food，无数据支撑。

**我们的解法**：
- 结构化日志（Bristol 分型 + 疼痛 + 急迫 + 饮食 + 症状标签）
- 关联分析 → 可视化「吃了 X 后 Y 小时内出现 Z 症状的概率」
- 个性化 trigger 排名 → 精准避雷

## 5. 产品架构

### 5.1 三条产品线

```
                    ┌──────────────────────┐
                    │      肠安地图        │
                    └──────────┬───────────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │

    │  A 线      │       │  B 线      │       │  C 线      │
    │  厕所地图  │       │  肠道日志  │       │  路线规划  │
    └───────────┘       └───────────┘       └───────────┘
         │                    │                    │
    · 浏览附近厕所      · Bristol 分型       · 起终点输入
    · 多维筛选          · 疼痛/急迫等级      · 沿途厕所匹配
    · 查看详情/评价     · 饮食记录           · 覆盖分析
    · 提交新厕所        · 症状标签           · 安全评分
    · 写评价            · 时间线             · 危险预警
                         · 关联洞察           · 保存/分享
```

### 5.2 数据闭环

```
用户出行 → 沿途使用厕所 → 实时验证厕所状态
                              ↓
用户记录日志 ← 关联厕所数据 ← 厕所数据更新
     ↓
发现规律 → 更好避雷 → 更多出行 → 更多数据贡献
```

## 7. 数据策略

### 7.1 数据来源

| 来源 | 获取方式 | 数量（当前） | 坐标系 | 更新频率 |
|---|---|---|---|---|
| OpenStreetMap | Overpass API | ~5000 | WGS-84 | 按需 |
| 高德 POI API | Place Search | ~200 | GCJ-02→WGS-84 | 按需 |
| 用户贡献 | 应用内提交 | 0 | WGS-84 | 实时 |
| 政府开放数据 | 数据门户下载 | 待接入 | 各异 | 月度 |

### 7.2 数据质量保障

```
多源交叉验证 → 重复去重（空间+名称） → 属性合并 → 
用户验证反馈 → 过期标记（6月无更新） → 自动化下架
```

### 7.3 隐私设计原则

- 🏠 **数据本地化**：肠道日志仅存用户自己的数据库中
- 👤 **匿名化**：默认开启，日志不关联个人身份
- 📤 **可导出**：一键 JSON 导出
- 🗑️ **可删除**：永久删除所有数据
- 🚫 **无追踪**：不使用第三方分析服务

- `meal_records` — 饮食记录
- `symptom_records` — 症状记录
- `routes` — 规划路线
